### 9.2 Rich Text — Quill.js v2

Integração completa do editor Quill Snow (via CDN `cdn.jsdelivr.net/npm/quill@2`) no modal de tarefas:

**Frontend (`public/assets/js/kanban.js`)**
- `initQuill()` — monta duas instâncias: `quillCreate` (modal de criação) e `quillDetail` (modal de detalhes)
- `closeCreateModal()` — chama `quillCreate.setContents([])` ao fechar para limpar o editor
- `openModal()` — descrição renderizada como HTML (`innerHTML`) em vez de `textContent`
- `toggleDescEditor(open)` — alterna entre modo leitura e edição da descrição; pre-popula Quill com HTML atual
- `saveDescription()` — envia `PATCH /api/tasks?id=X` com HTML do editor; atualiza view inline sem reload
- `closeModal()` — chama `toggleDescEditor(false)` para resetar estado do editor ao fechar

**Template (`templates/default/pages/kanban.php`)**
- Modal de criação: `<textarea>` substituído por `<div id="create-desc-editor" class="quill-editor">` + `<input type="hidden" name="description">`
- Modal de detalhes: seção de descrição com alternância leitura/edição:
  - Modo leitura: `<div id="modal-desc">` renderiza HTML rico
  - Modo edição: `<div id="desc-editor-wrap">` com `<div id="modal-desc-editor">` + botões Salvar/Cancelar
  - `id="save-desc-btn"` adicionado ao botão Salvar para controle de estado desabilitado
- CSS Quill adicionado ao `extra_css` da rota `/boards` em `routes/web.php`

**Estilos (`public/assets/css/kanban.css`)**
```css
.quill-editor .ql-toolbar   { border-bottom: 1px solid #f1f5f9; }
.quill-editor .ql-container { border: none; font-family: 'Inter'; }
#modal-desc img             { max-width: 100%; border-radius: .5rem; }
```

## 10. Sessão 20/03/2026 (cont.) — Avatares reais + Lookup de Responsável

### 10.1 Avatares reais em `projects.php`

Removidos completamente os avatares dicebear (`api.dicebear.com/7.x/avataaars/svg?seed=Anna/Bob`).

**Solução:** JS inline busca `GET /api/users` ao carregar a página e renderiza iniciais coloridas para cada membro da empresa:
- Máximo de 3 avatares visíveis + badge `+N` para o restante
- Cores cicladas por índice (indigo, verde, laranja, roxo, azul, amarelo)
- Cada avatar exibe o nome completo via `title`

### 10.2 Lookup de responsável (`assigned_to`)

**Backend — `GET /api/users`** (novo endpoint em `routes/api.php`):
- Retorna `[{id, name, email}]` para todos os usuários ativos da empresa autenticada
- Filtro por `company_id` da sessão + `status = 'active'`, ordenado por nome

**`ProjectController`** (`app/Controllers/ProjectController.php`):
- Injetado `SessionStore` via construtor (parâmetro opcional `?SessionStore $session`)
- Removido `$companyId = 1` e `$userId = 1` hardcoded em `index()` e `create()`
- `routes/web.php` e `routes/api.php` atualizados para passar `$session` ao construtor

**Frontend — `kanban.js`**:
- Nova variável global `companyUsers = []` — carregada uma vez no boot
- `loadCompanyUsers()` — busca `/api/users` e popula `companyUsers`; chamada assíncrona antes de `fetchAllTasks()`
- `userById(id)` — helper para lookup por ID
- `userInitials(name)` — gera 2 iniciais do nome
- `populateUserSelects()` — preenche `#create-assignee` e `#meta-assignee` com opções reais
- `renderTask()` — exibe iniciais do responsável no avatar do card (em vez de `#id`)
- `updateAssignee(userId)` — `onchange` do `#meta-assignee`; chama `PATCH /api/tasks?id=X` e refresca o board

**Frontend — `kanban.php`**:
- Modal de criação: campo `Responsável` (`<select name="assigned_to" id="create-assignee">`) ao lado do `Prazo`
- Modal de detalhes: `#meta-assignee` convertido de `<div>` para `<select onchange="updateAssignee()">` — troca de responsável direto da sidebar sem abrir outro modal

**Payload de criação (`kanban.js` form submit)**:
- `assigned_to` vazio → removido do payload (`delete data.assigned_to`)
- `assigned_to` preenchido → convertido para `int`
