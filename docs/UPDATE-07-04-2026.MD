# Registro de Alterações — 07/04/2026

## Resumo
- Secrets por projeto agora suportam **Título** e **Descrição** (UI + API + persistência).
- Migrations de `project_secrets` atualizadas (MySQL e SQLite) para incluir `title` e `description`.
- Pequenos ajustes visuais no Kanban e em ícones, além de alternância de visibilidade de senha no login.

## Secrets — Título e Descrição
- UI:
  - Modal de criação/edição passou a ter campos `title` e `description`.
  - Tabela passa a exibir “Título/Chave” e “Descrição”.
- API:
  - `GET /api/project-secrets?project_id={id}` agora retorna: `title`, `description`.
  - `POST /api/project-secrets` aceita: `{ project_id, secret_key, secret_value, title?, description? }`
  - `PATCH /api/project-secrets?id={secretId}` aceita: `{ secret_key, secret_value, title?, description? }`
- Validação:
  - `title` opcional (máximo 190 chars).

## Banco de Dados
- Schema:
  - `project_secrets.title VARCHAR(190) NULL`
  - `project_secrets.description TEXT NULL`
- Nota para bases já existentes:
  - Caso a tabela `project_secrets` já exista, aplicar `ALTER TABLE` para adicionar colunas em vez de recriar a tabela.

## UI — Ajustes
- Kanban:
  - Ajustes de tons de cinza em labels/mensagens e placeholders para melhorar contraste.
- Login:
  - Botão para alternar visibilidade da senha.
- Projetos:
  - Ações “Membros” e “Secrets” com botões/ícones consistentes.
